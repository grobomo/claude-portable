# Data Model: Spec Kit + GSD Integration

## Entities

### BridgeRequest
Source: `rone-boothapp-bridge/requests/pending/<id>.json`
```
{
  sender: string          # "Joel Ginsberg (local)" or Teams display name
  text: string            # Raw task description
  context: string[]       # Last 25 chat messages (may be empty)
  timestamp: ISO8601      # When submitted
  chat_id: string         # "local" or Teams chat ID
}
```
State transitions: `pending/` -> `dispatched/` -> `completed/` | `failed/`

### SpecArtifacts
Generated by: dispatcher via `spec-generate.sh`
Location: temp dir, then SCP'd to worker at `/workspace/boothapp/.specs/<slug>/`
```
.specs/<slug>/
  spec.md     # Problem, solution, components, success criteria
  plan.md     # Technical approach, file paths, dependency order
  tasks.md    # Ordered checkbox tasks
```

### GSDScaffold
Generated by: dispatcher via `spec-generate.sh` (lines 82-94)
Location: SCP'd to worker at `/workspace/boothapp/.planning/`
```
.planning/
  config.json   # { mode: "yolo", depth: "quick", auto_initialized: true, spec_dir: ".specs/<slug>" }
  quick/        # Empty — worker creates task dirs here
```

### GSDTaskDir
Created by: worker (after GSD gate blocks first tool use)
Location: `/workspace/boothapp/.planning/quick/NNN-<slug>/`
```
.planning/quick/001-<slug>/
  001-PLAN.md     # Goal + success criteria (from spec)
  001-SUMMARY.md  # Post-implementation verification
```

### WorkerResult
Created by: dispatcher after worker completes
Location: `rone-boothapp-bridge/requests/completed/<id>.json`
```
{
  ...original request fields...,
  result: string        # Worker stdout (truncated to 5000 chars)
  completed_at: ISO8601
  worker: string        # Worker name
}
```

## Data Flow

```
1. BridgeRequest created in pending/
   |
2. Dispatcher polls, moves to dispatched/
   |
3. Dispatcher calls spec-generate.sh with request.text
   |  -> creates SpecArtifacts in /tmp/spec-<id>/
   |  -> creates GSDScaffold in /tmp/spec-<id>/
   |
4. Dispatcher SCP's artifacts to worker host
   |  -> docker cp into /workspace/boothapp/
   |
5. Dispatcher SSH's to worker: claude -p with spec preamble + original prompt
   |
6. Worker's GSD gate blocks tool execution
   |  -> Worker reads .specs/<slug>/spec.md
   |  -> Worker creates GSDTaskDir with 001-PLAN.md
   |  -> GSD gate passes, worker implements
   |
7. Worker creates PR against altarr/boothapp
   |
8. Dispatcher captures stdout, moves request to completed/
```

## Validation Rules

- `spec-generate.sh` MUST produce at minimum a `spec.md` (fallback stub if claude -p fails)
- GSD `config.json` MUST have `auto_initialized: true` for gate to activate
- PLAN.md filename MUST match pattern `NNN-PLAN.md` where NNN matches the dir prefix
- Worker prompt MUST include spec preamble when spec generation succeeds
