# Log Gotchas on Stop

Before stopping, review what went wrong or was surprising in this session. If any gotchas were encountered (unexpected behavior, workarounds needed, things that wasted time), append them to the project's `.claude/rules/` as a rule file so future sessions don't repeat the mistake.

Format: one file per topic in `.claude/rules/`, one bullet per gotcha. If the gotcha is global (applies to all projects), put it in `~/.claude/rules/` instead.
